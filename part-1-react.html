React - What is it

  • React is a declarative, efficient, and flexible JavaScript library for building user interfaces.
  • In a complete web application, React represents the view layer only - it renders HTML and handles events (no business logic - that's where Redux comes in!)

React - What is it

In contrast to 'separation of concerns'/MVC templating, views (HTML) are contained in the Javascript.

React - What is it

What can you do with React?

  • create reusable HTML components
  • attach interactivity to HTML components with event handlers
  • see your view logic alongside your HTML templates
  • utilize the full power of the Javascript language for rendering components

Benefits of React

Virtual DOM

  • React keeps a 'virtual' copy of the DOM in memory
  • When state changes in the application, first the changes are applied to the virtual DOM
  • React diffing engine handles re-renders of only the smallest necessary changes to the actual DOM
  • Instead of keeping a list of values to 'watch' (the way Angular or Backbone does), React re-renders DOM elements based on this DOM-virtual DOM comparison.

Facebook's create-react-app

A quick word on Webpack & build tools

create-react-app uses Webpack - a powerful tool to compile all your files into one neat "bundle".

  • Takes care of transpiling ES6, JSX - so you can write the most up-to-date version of Javascript & React that some older browsers don't support.
  • Imports all your CSS/SASS, images, and fonts
  • Requires myriad configuration options, but create-react-app picks common ones that allow us to get up and running quickly.
  • GDI Philly has a class on build tools for more information.

npm

  • npm (node package manager) allows us to use code published by other developers.
  • Use npm install my-package-name to install a package.
  • External dependencies are registered in package.json at the root of your project - so anyone trying to run your code has a record of all of its dependencies
  • For this project, we need to use an npm module called create-react-app

JSX: What is it?

  • JSX is a preprocessor step that adds XML syntax to JavaScript.
  • You can definitely use React without JSX but JSX makes React a lot more elegant.
  • Just like XML, JSX tags have a tag name, attributes, and children.
(Source)

JSX features

JSX looks a lot like HTML, but uses some of its own syntax & attribute names

  • className instead of class (because class is a reserved keyword in Javascript)
  • Event names: onClick, onKeyUp, onChange take care of cross-browser compatibility

JSX Under the Hood

Literal vs. Interpreted Props

Some props take literal values like strings or numbers

  • className="app-header"
  • itemCount=3


    • If the value of the prop is Javascript code (including a function reference or anonymous function), it needs to be wrapped in brackets: {}

    • text={`Today's date is ${date}.`}
    • onClick={this.clickHandler}
    • onMouseEnter={() => console.log("MOUSE ENTERED"); }

State

Use state when your component needs to keep track of something

  • Value of a text input
  • Is a menu open or closed?
  • The items in a grocery list app
  • The response from the Meetup API

Setting Up State

  • Each React component has a constructor function that gets called before it renders - here's where your initial state should be set.
  • Bonus: Remember Object-oriented programming? Our new component will inherit from React's Component prototype, and get custom methods and properties that we define.

class App extends Component {
  constructor(props) {
    super();
    this.state = {
       meetups: []
    }
  }
}
