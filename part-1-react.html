React - What is it

  • React is a declarative, efficient, and flexible JavaScript library for building user interfaces.
  • In a complete web application, React represents the view layer only - it renders HTML and handles events (no business logic - that's where Redux comes in!)

React - What is it

In contrast to 'separation of concerns'/MVC templating, views (HTML) are contained in the Javascript.

React - What is it

What can you do with React?

  • create reusable HTML components
  • attach interactivity to HTML components with event handlers
  • see your view logic alongside your HTML templates
  • utilize the full power of the Javascript language for rendering components

Redux - What is it?

Redux is a predictable state container for JavaScript apps.

Redux - What makes it special?

  • One-way data flow
  • Single data store describes entire application state
  • Instead of directly modifying independent data 'models', any change to the application describes an alteration of the state tree

Benefits of React

Virtual DOM

  • React keeps a 'virtual' copy of the DOM in memory
  • When state changes in the application, first the changes are applied to the virtual DOM
  • React diffing engine handles re-renders of only the smallest necessary changes to the actual DOM
  • Instead of keeping a list of values to 'watch' (the way Angular or Backbone does), React re-renders DOM elements based on this DOM-virtual DOM comparison.

    Props

    What type of item is each prop?

    Source: props.js

    Props: solution

    • className: string
    • onClick: function
    • labelAltText: interpolated string
    • onMouseEnter: function
    • color: string

    Literal vs. Interpreted Props

    Some props take literal values like strings or numbers

    • className="app-header"
    • itemCount=3


      • If the value of the prop is Javascript code (including a function reference or anonymous function), it needs to be wrapped in brackets: {}

      • text={`Today's date is ${date}.`}
      • onClick={this.clickHandler}
      • onMouseEnter={() => console.log("MOUSE ENTERED"); }

    Proptypes

    Proptypes tell the component what props it might receive, their types, and whether they are required.

    • Helpful for reference working with a component so you know if "items" is an array, string, number, etc...
    • The browser console will warn you if a component receives an unexpected prop, or doesn't receive a required one - great for debugging!

    Style: a special prop

    React components accept a special style prop, which can be used in place of SASS/CSS stylesheets.

    Note:

    • camel-cased properties
    • doesn't take care of browser prefixes
    • need {} for JavaScript evaluation, with
      a style object { color: 'blue', fontSize: 16 } inside
